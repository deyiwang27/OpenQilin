# Funding and Sponsorship Strategy

Date: `2026-03-14`
Status: `discussion draft`
Stage: `pre-kickoff`

## 1. Purpose

- Define a realistic funding and sponsorship strategy for OpenQilin.
- Split the strategy into:
  - MVP-v2-oriented preparation
  - longer-term sponsorship and funding development
- Keep the strategy aligned with OpenQilin's open-source-community-oriented direction.

## 2. Framing

OpenQilin is built for the solopreneur: turning one capable person into a coordinated AI-augmented team.
It does this through governed role delegation, project-centered execution, and explicit control over authority, budget, and evidence.

OpenQilin is also intended to be open-source-community-oriented.

That means funding strategy should not be treated only as:
- cloud credits
- free API access
- sponsor logos

It should also support:
- product credibility
- community growth
- contributor trust
- early sustainability

## 3. Strategic Conclusion

The most realistic near-term support paths are:
- startup cloud-credit programs
- GitHub Sponsors / OSS funding
- startup ecosystem programs and partner perks

The least reliable assumption is:
- expecting direct custom sponsorship or broad free API support from OpenAI or Google just because the project is interesting

So the strategy should be practical:
- prepare OpenQilin so it is eligible and credible
- apply to realistic programs
- use community-oriented assets to strengthen the case

## 4. Part I: MVP-v2-Oriented Strategy

## 4.1 Goal

By the end of MVP-v2, OpenQilin should be ready enough that:
- it can be presented publicly
- it can apply credibly to startup and sponsorship programs
- it can begin attracting community support and possibly GitHub Sponsors

I agree with the position that MVP-v2 should explicitly prepare for this.

## 4.2 MVP-v2 sponsorship objective

MVP-v2 should prepare OpenQilin to be "funding-ready" and "sponsorship-ready," not just technically working.

That means MVP-v2 should intentionally leave the project with these assets ready:
- public website with matching domain email
- sharp project thesis
- one-page deck
- MVP-v2 demo
- clear repo README
- roadmap
- explanation of why OpenQilin matters

I think this is the right objective.

## 4.3 Why these assets matter

These are not just marketing materials.

They are the minimum package most startup-credit or sponsor conversations need in practice:
- website and domain email make the project/company look real
- thesis and "why it matters" create narrative clarity
- demo and README prove the product exists
- roadmap shows seriousness and direction
- deck makes outreach efficient

Without these, even a technically interesting project tends to look too early or too informal for support programs.

## 4.4 MVP-v2 deliverables that support funding readiness

### A. Public website and matching domain email

Purpose:
- basic external legitimacy
- a stable landing page for the product thesis, demo, and contact

Minimum expectation:
- simple site explaining what OpenQilin is, who it is for, and current status
- matching project/domain email for applications and outreach

### B. Sharp project thesis

Required message:
- OpenQilin is built for the solopreneur: turning one capable person into a coordinated AI-augmented team.
- It does this through governed role delegation, project-centered execution, and explicit control over authority, budget, and evidence.

Purpose:
- gives the project a distinct, fundable identity
- prevents it from being perceived as "another vague AI agents repo"

### C. One-page deck

Purpose:
- concise startup/sponsor conversation tool

Suggested structure:
- problem
- target user
- why now
- OpenQilin approach
- MVP-v2 scope
- demo/screens
- roadmap
- what support is needed

### D. MVP-v2 demo

Purpose:
- prove the product is real

Suggested demo path:
- owner initiates or advances a governed project
- PM coordinates execution
- Secretary explains status/alerts
- dashboard shows progress, budget/cost, and system trust signals

### E. Clear repo README

Purpose:
- convert curiosity into understanding
- help both sponsors and contributors quickly evaluate the project

Minimum expectation:
- product thesis
- who it is for
- what works now
- quickstart
- demo/screenshots
- roadmap/status

### F. Roadmap

Purpose:
- show there is a plan beyond the demo

Should distinguish:
- MVP-v2 scope
- post-MVP-v2 direction
- open questions

### G. Explanation of why OpenQilin matters

Purpose:
- make the project legible beyond code

Core framing:
- solo operators need leverage
- they do not just need another chatbot
- they need a governed AI operating model they can trust

## 4.5 Realistic MVP-v2 funding targets

The most realistic near-term targets are:

### A. Google Cloud startup support

Google Cloud currently offers:
- `Start tier` for pre-funded startups with `$2,000` in credits
- `Scale tier` for early-stage funded startups with `$200,000`
- up to `$350,000` for qualified AI-first startups

OpenQilin is more likely to fit early-stage or pre-funded support than a custom sponsorship story.

### B. AWS Activate

AWS currently offers:
- `Founders`: `$1,000`
- `Portfolio`: up to `$100,000`

This is realistic if OpenQilin becomes company-structured enough to apply.

### C. Microsoft for Startups / Azure startup credits

Microsoft currently offers:
- Azure startup credit offer up to `$5,000`
- Microsoft for Startups investor-network paths with higher Azure credits

### D. GitHub Sponsors

GitHub Sponsors is the most direct OSS-friendly support path.

It is useful because:
- it aligns with community orientation
- it supports maintainers directly
- it can start small

### E. OpenAI-related opportunities

Current conclusion:
- do not assume a general OpenAI startup-credits path exists
- treat OpenAI opportunities as program/network opportunities when relevant, not as the core funding assumption

OpenAI Grove exists, but it is not a general startup credits program and is not the same as direct infrastructure sponsorship.

## 4.6 MVP-v2 preparation checklist

By the end of MVP-v2, OpenQilin should aim to have:

- public website with matching domain email
- sharp project thesis
- one-page deck
- MVP-v2 demo
- clear repo README
- roadmap
- explanation of why OpenQilin matters
- example config and quickstart
- contribution guide
- public-facing screenshots or demo assets

This should be treated as part of MVP-v2 readiness, not merely as later polish.

## 5. Part II: Longer-Term Funding and Sponsorship Strategy

## 5.1 Goal

Longer term, OpenQilin should move from "eligible for credits" to:
- fundable
- sponsorable
- community-supported
- sustainable enough to keep building

## 5.2 Longer-term strategy pillars

### A. Infrastructure support

Continue pursuing:
- cloud credits
- partner/perk programs
- startup ecosystem support

But do not build the long-term project economics on free credits alone.

### B. Community-backed support

Grow:
- GitHub Sponsors
- public supporters
- community donations or maintainer support

This is especially aligned with the repo's open-source-community orientation.

### C. Strategic partnerships

Over time, OpenQilin may become interesting for:
- AI infrastructure partners
- startup programs
- cloud vendors
- adjacent tooling ecosystems

But this becomes more realistic only after:
- clear user traction
- strong public demos
- coherent product identity

### D. Sustainable product narrative

The project should remain easy to explain:
- a governed AI operating system for solopreneurs
- not a generic multi-agent experiment

That clarity improves sponsor confidence.

### E. Community trust

Long-term sponsorability depends on:
- maintained docs
- visible progress
- active issue triage
- healthy contributor experience
- repo credibility

## 5.3 Longer-term funding mix

The likely healthy mix is:
- startup credits early
- OSS sponsor support in parallel
- later strategic partnerships if the project gains traction

This is much stronger than relying on any one provider's generosity.

## 6. Bottom Line

Yes, it makes sense to make funding/sponsorship readiness part of MVP-v2.

I agree with making these explicit MVP-v2 objectives:
- public website with matching domain email
- sharp project thesis
- one-page deck
- MVP-v2 demo
- clear repo README
- roadmap
- explanation of why OpenQilin matters

That is the right preparation layer if OpenQilin wants to:
- apply for startup programs
- approach sponsors credibly
- grow an open-source community
