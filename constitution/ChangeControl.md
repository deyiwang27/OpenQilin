# Change Control

## Scope
Defines how constitutional artifacts are modified.

## Approval Policy
- Rule text changes require Owner approval.
- Threshold changes require Owner approval.
- Non-functional metadata updates may be approved by CEO.

## Process
1. Propose change with rationale and impact.
2. Link affected rule IDs.
3. Run conformance review.
4. Approve and increment policy version.
5. Snapshot to `constitution/versions/`.

## Audit Requirement
All policy changes must be logged with author, approver, timestamp, and diff summary.
