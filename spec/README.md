# OpenQilin Spec Layer

Purpose: implementation-level specifications for AI engineering agents.

## Scope
- Normative system behavior.
- Interfaces, state machines, and rule IDs.
- Conformance requirements.

## Canonical Documents
- `Governance.md`
- `Constitution.md`
- `AgentLifecycle.md`
- `Communication.md`
- `Memory.md`
- `Budget.md`

## Precedence
1. `constitution/` (runtime institutional source of truth)
2. `spec/` (implementation contract)
3. `docs/` (human-facing guidance)

## Writing Rules
- Use MUST/MUST NOT for enforceable statements.
- Keep rule IDs stable.
- Include testable acceptance criteria.
