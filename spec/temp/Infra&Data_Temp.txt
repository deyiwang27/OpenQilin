	1. Memory Stores
	Memory = Scope × Lifecycle
	Storage lifecycle
		• Working Memory Store(hot)
		• Project Memory Store(warm)
		• Archive Memory Store (cold)
	Scope dimension:
		• Agent
		• Project
		• Global
	Structured Index Filters:
		• project_id
		• agent_id
		• memory_scope (hot/warm)
		• memory_type (decision, summary, alert, discussion)
		• timestamp
		• status
		• version

	Change Data Capture (CDC): Relational DB → Vector DB
	Ensure Memory Stores (vector DB) are always consistent with authoritative project/agent state.
	Any insert/update in relevant tables (Project Container, Logs, Agent Registry) triggers CDC events.
	Direct ETL scripts: Python listener on PostgreSQL triggers → push updates to vector DB.
		• Hot memory updated in near real-time.
		• Warm memory updated periodically, e.g., nightly summarization.
		• Cold archive updated weekly/monthly.
	
	Memory Mapping
	DB Table / Event	Vector DB Memory Level	Notes
	Project updates / logs	Hot Memory	Latest context for active reasoning
	Execution logs aggregation	Warm Memory	Summarized history, compressed
	Snapshots / old logs	Cold Archive	Periodic snapshot, read-only
